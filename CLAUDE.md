## Основная задача

Образовательный ресурс о Bitcoin. **Только Bitcoin — никаких других тем.**
Весь контент структурируется по единой схеме. Архитектура и дизайн → [STRUCTURE.md](STRUCTURE.md)

---

## Правила структурирования

Любые данные, графики, тексты — разбирать по схеме:
- **Что это** — краткое определение
- **Контекст** — исторические аналогии
- **Что означает сейчас** — интерпретация
- **Важные оговорки** — риски, ограничения

### Категории

| Категория | Примеры |
|-----------|---------|
| 📊 On-chain | Production Cost, MVRV, NVT |
| 📈 Технический анализ | Уровни, паттерны, тренды |
| 🏦 Макроэкономика | Ставки, ликвидность, DXY |
| ⛏️ Майнинг | Хешрейт, сложность, капитуляция |
| 📰 Нарратив | Регуляция, институционалы, события |
| 🔗 Layer 2 | Stacks, Lightning, сайдчейны |
| 🏛️ Владение | Распределение по типам владельцев, институционализация |

## Правило наполнения вкладок

> Новые вкладки наполняются **строго последовательно** — по одной за раз.
> Claude **НЕ** наполняет вкладку из собственных знаний по своей инициативе.
>
> 1. Пользователь присылает материал
> 2. Claude структурирует по схеме выше
> 3. Переход к следующей теме — только после явного сигнала пользователя

Накопленные материалы и база сигналов → [SIGNALS.md](SIGNALS.md)

## Инструкция для Claude

При добавлении нового сигнала — **обязательно оба места**:
1. Добавить объект в массив `SIGNALS` в `index.html` (сайт)
2. Добавить запись в [SIGNALS.md](SIGNALS.md) с датой и категорией (документация)

> ⚠️ Только SIGNALS.md без index.html = сигнал не появится на сайте.

При добавлении новой информации:
1. Структурировать по схеме выше
2. Связать с уже имеющимися данными, если есть пересечения

При изменении этого файла:
1. Поднять версию в шапке (+0.1 минор / +1.0 мажор)
2. Обновить дату
3. Добавить строку в [CHANGELOG.md](CHANGELOG.md)
