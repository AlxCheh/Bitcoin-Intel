## Добавление сигнала

Каждый новый сигнал — **два файла одновременно**:

**1. `signals.json`** — объект в массив:
```json
{
  "date": "YYYY-MM-DD",
  "cat": "narrative",
  "catLabel": "📰 НАРРАТИВ",
  "dir": "neg",
  "horizon": "short",
  "theme": "institutionalization",
  "weight": "media",
  "actor": "etf",
  "flow": "outflow",
  "signal": "Заголовок сигнала",
  "data": ["метрика 1", "метрика 2"],
  "context": "Исторический контекст и аналогии.",
  "caveat": "Риски и ограничения интерпретации.",
  "source": "Источник (месяц год)"
}
```

### Метаметки сигнала

**`dir`** — что сигнал говорит о силе Bitcoin как актива в своём горизонте:

| Значение | Смысл |
|----------|-------|
| `pos` | усиливает позицию BTC |
| `neg` | создаёт давление на BTC |
| `neu` | структурный факт без направления |

**`horizon`** — временной горизонт:

| Значение | Смысл |
|----------|-------|
| `short` | недели–месяцы |
| `mid` | месяцы–год |
| `long` | годы, структурные изменения |

**`theme`** — смысловой кластер (группирует сигналы в глобальную картину):

| Значение | Смысл |
|----------|-------|
| `supply` | дефицит, эмиссия, потерянные монеты |
| `institutionalization` | ETF, корпорации, правительства, владение |
| `infrastructure` | Lightning, Layer 2, майнинг |
| `macro` | ставки, ликвидность, макро-давление |
| `narrative` | философия, регуляция, нарративы |

**`weight`** — достоверность источника:

| Значение | Смысл |
|----------|-------|
| `onchain` | верифицируемые данные блокчейна |
| `market` | биржевые данные, цена, объёмы |
| `media` | медиа, аналитики, оценки |
| `primary` | первичный источник (высказывание, документ) |

**`actor`** — субъект сигнала:

| Значение | Смысл |
|----------|-------|
| `etf` | биржевые фонды (BlackRock, Fidelity) |
| `corporate` | публичные компании (Strategy, MSTR) |
| `government` | государственные резервы |
| `defi` | протоколы, on-chain структуры |
| `retail` | розничные инвесторы |
| `miner` | майнеры |

**`flow`** — направление капитала:

| Значение | Смысл |
|----------|-------|
| `inflow` | капитал входит в BTC |
| `outflow` | капитал уходит из BTC |
| `neutral` | движения капитала нет или неизвестно |

**2. `SIGNALS.md`** — текстовая запись с датой и категорией.

> `index.html` для сигналов не трогать — данные загружаются из `signals.json` через fetch.

## Схема структурирования материала

Любой входящий материал разбирать по четырём пунктам:
- **Что это** — краткое определение
- **Контекст** — исторические аналогии
- **Что означает сейчас** — интерпретация
- **Важные оговорки** — риски, ограничения

### Категории сигналов

| cat | catLabel | Примеры |
|-----|----------|---------|
| `onchain` | 📊 On-chain | Production Cost, MVRV, NVT |
| `ta` | 📈 Технический анализ | Уровни, паттерны, тренды |
| `macro` | 🏦 Макроэкономика | Ставки, ликвидность, DXY |
| `mining` | ⛏️ Майнинг | Хешрейт, сложность, капитуляция |
| `narrative` | 📰 Нарратив | Регуляция, институционалы, события |
| `layer2` | 🔗 Layer 2 | Stacks, Lightning, сайдчейны |
| `ownership` | 🏛️ Владение | Распределение по типам владельцев |

## Правило наполнения вкладок

Claude **не наполняет вкладки из собственных знаний**. Только из материалов пользователя.

1. Пользователь присылает материал
2. Claude структурирует по схеме выше
3. Переход к следующей теме — только после явного сигнала пользователя
