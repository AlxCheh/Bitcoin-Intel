## Основная задача

Вся информация в этом проекте — существующая и новая — должна быть **структурирована**.

---

## Правила структурирования информации

### При получении новых материалов
- Любые данные, графики, тексты, индикаторы — разбирать по единой схеме:
  - **Что это** — краткое определение
  - **Контекст** — исторические аналогии, предыдущие случаи
  - **Что это означает сейчас** — интерпретация текущей ситуации
  - **Важные оговорки** — риски, ограничения индикатора

### Формат подачи
- Использовать **таблицы** для исторических сравнений
- Использовать **заголовки** для разделения смысловых блоков
- Ключевые цифры и термины выделять **жирным**
- Эмодзи — умеренно, только для навигации

## Структура приложения — кластеры навигации

Двухуровневая навигация: 3 кластера с выпадающими меню.

| Кластер | Вкладки сейчас | Запланировано |
|---------|----------------|---------------|
| 🟠 **LIVE** (реальное время) | ОБЗОР, АНАЛИТИКА, ПУЛЫ | МЕМПУЛ, РЫНОК |
| 📘 **ЗНАНИЕ** (стабильная теория) | ОСНОВЫ, ТЕХНОЛОГИИ | БЕЗОПАСНОСТЬ |
| 📜 **КОНТЕКСТ** (прошлое) | ИСТОРИЯ | — |

**Критерий распределения:** LIVE = меняется в реальном времени, ЗНАНИЕ = стабильная теория, КОНТЕКСТ = прошлое.

## Правило наполнения вкладок

> Новые вкладки наполняются **СТРОГО последовательно** — по одной за раз.
> Claude **НЕ** наполняет вкладку контентом из собственных знаний или веб-поиска по своей инициативе.
>
> Порядок работы с темой:
> 1. Пользователь предоставляет исходный материал (текст, данные, скриншот)
> 2. Claude структурирует его по схеме «Что это / Контекст / Что означает / Оговорки»
> 3. Claude углубляет и прорабатывает тему **только на основе этого материала**
> 4. Переход к следующей теме — только после явного сигнала пользователя
>
> **Очередь наполнения** (по запросу, в этом порядке): МЕМПУЛ → РЫНОК → БЕЗОПАСНОСТЬ

## Формат базы сигналов (единый на весь проект)

> Любой новый рыночный/нарративный материал оформляется как **карточка-сигнал** — запись в массиве `SIGNALS` (в index.html).
>
> Поля записи:
> - `date` — дата (YYYY-MM-DD)
> - `cat` / `catLabel` — категория (macro/narrative/onchain/mining + эмодзи-метка)
> - `dir` — направление для BTC: `neg` 🔴 / `pos` 🟢 / `neu` ⚪
> - `signal` — суть одной фразой
> - `data[]` — ключевые цифры (чипы)
> - `context` — предыстория, аналогии
> - `caveat` — риски, оговорки, чего не знаем
> - `source` — источник
>
> Сводка и фильтры строятся автоматически из данных. Добавить сигнал = добавить объект в массив.
