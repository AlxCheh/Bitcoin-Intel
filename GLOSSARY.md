## Аналитические термины

**Signal** (Сигнал)  
Единица наблюдения за рынком Bitcoin. Структурированный объект с уникальным `id`, описывающий одно событие или факт с его интерпретацией (`context`, `caveat`), направлением (`dir`), горизонтом (`horizon`) и ролью в нарративе (`narrative_role`). Первичный атом базы знаний.

**Cluster** (Кластер)  
Группа сигналов, объединённых общим `tension` — противоречием между двумя силами. Кластер создаётся когда появляются 2+ сигнала с общим противоречием которого нет в существующих. Например: `strategy_model_stress`, `etf_institutional_flow`.

**Tension** (Противоречие)  
Краткая формулировка столкновения двух сил внутри кластера. Всегда по конструкции «X vs Y» / «X несмотря на Y» / «X при условии что Y». Начинается с заглавной буквы. Не описание факта — формулировка противоречия. Отображается золотой полосой в блоке «Главные нарративы».

**macro_implication** (Макроимпликация)  
Ответ на вопрос «что это меняет для BTC структурно». Не пересказ события — вывод об изменении в системе. Формирует основной текст карточки нарратива.

**narrative_role** (Роль в нарративе)  
Позиция сигнала в логической цепочке кластера. Четыре значения:
- `trigger` — первый сигнал кластера или резкий перелом
- `complication` — добавляет противоречие в существующий нарратив
- `resolution` — закрывает tension предыдущих сигналов
- `background` — структурный контекст без острого события

**Synthesis** (Синтез)  
Итоговая карточка нарратива для кластера: tension + macro_implication + takeaway + strength. Строится алгоритмом из сигналов кластера. Может быть одобрена вручную (помечается меткой «✓ Аналитик») или рассчитана автоматически («◈ Алгоритм»).

**strength** (Сила нарратива)  
Оценка убедительности кластера: `strong` / `moderate` / `weak`. Определяется суммарным score кластера.

**score** (Оценка кластера)  
Числовая оценка значимости кластера для отображения в топ-4 нарративов. Складывается из: freshness (+3/сигнал за 7 дней), contradicts (+5/сигнал), tension (+2/сигнал), onchain weight (+4/сигнал), trigger role (+4/сигнал).

**mNAV** (Market NAV)  
Рыночная стоимость компании / стоимость её BTC-резервов. Коэффициент < 1.0 означает дисконт к стоимости BTC на балансе (рынок оценивает компанию дешевле чем её биткоины).

**window_days**  
Временное окно для расчёта score кластера. Сигналы вне окна не влияют на freshness, но остаются в базе.

**anchor-сигнал**  
Сигнал с максимальным количеством `links.contradicts` в кластере. Его `tension` отображается в карточке нарратива как главный крючок.

## Технические термины

**PYTHONHASHSEED**  
Переменная окружения Python, контролирующая детерминизм `hash()`. При `PYTHONHASHSEED=random` (дефолт) — `hash()` возвращает разные значения в разных процессах. Для воспроизводимости синтеза: `PYTHONHASHSEED=0`.

**partA / partB**  
Части нарративного текста синтеза. `partA` — тезис (что происходит), `partB` — импликация (что это означает структурно). Разделитель — символ ` · `.

**bridge**  
Связующая фраза между `partA` и `partB` в тексте синтеза. Выбирается функцией `select_bridge(phase, seed)` по фазе рынка. Примеры: «создаёт структурную основу для», «при этом рынок».

**signal_strength**  
Агрегированный показатель силы сигнала в контексте кластера. Учитывает `weight`, `horizon`, наличие `tension` и `links.contradicts`.

**Aggregate Root**  
Корневая сущность в Domain Model, через которую происходит доступ ко всем связанным объектам. В Bitcoin Intel — `Signal` является Aggregate Root для своих связей.

**append-only**  
Принцип работы с данными: записи только добавляются, не изменяются и не удаляются. Применяется к `data/events.jsonl` (audit trail) и `signals.json` (архив сигналов).

**synthesis_cache**  
Производный артефакт: предрассчитанные синтезы для каждого кластера. Хранится в `data/synthesis_cache.json`. При изменении данных пересчитывается скриптом `scripts/rebuild_cache.py`.

**LEGACY_LINKS_ENABLED**  
Флаг переходного периода. При `True` — система читает связи из обоих источников: устаревшего `links.*` в `signals.json` и нового `relationships.json`. После завершения миграции — `False`.

**fallback**  
Резервное значение при отсутствии данных. Например: если у кластера нет синтеза — отображается заглушка `renderWeakSignalPlaceholder`.

## Поля объекта Signal

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | string | Уникальный идентификатор: `PREFIX-YYYY-MMDD-NNN` |
| `date` | string | Дата события в UTC, формат `YYYY-MM-DD` |
| `cat` | string | Категория: `onchain`, `ta`, `macro`, `mining`, `narrative`, `layer2`, `ownership` |
| `catLabel` | string | Метка категории с эмодзи для отображения |
| `dir` | string | Направление: `pos` / `neg` / `neu` |
| `horizon` | string | Горизонт: `short` / `mid` / `long` |
| `theme` | string | Смысловой кластер: `supply`, `institutionalization`, `infrastructure`, `macro`, `narrative` |
| `weight` | string | Достоверность: `onchain` / `market` / `media` / `primary` |
| `actor` | string | Субъект: `etf`, `corporate`, `government`, `defi`, `retail`, `miner` |
| `flow` | string | Капитал: `inflow` / `outflow` / `internal` / `neutral` |
| `signal` | string | Заголовок сигнала |
| `data` | array | Ключевые метрики |
| `context` | string | Исторический контекст |
| `caveat` | string | Риски и ограничения |
| `source` | string | Источник и дата публикации |
| `links` | object | Связи: `confirms`, `contradicts`, `context_chain` |
| `narrative_role` | string | Роль: `trigger` / `complication` / `resolution` / `background` |
| `cluster` | string | Идентификатор кластера |
| `tension` | string | Формулировка противоречия |
| `macro_implication` | string | Структурный вывод |
