# BAMS Architecture Review (BAR) v1.0

**Объект:** BAMS v1.1 → v1.2 · **Авторитетный вход:** BDKS v1.2 (заморожен для данной задачи)
**Метод:** контролируемая архитектурная трансформация — BDKS определяет знание, BAMS определяет метод; ни один элемент знания не существует в BAMS без трассы к BDKS.

**Интерпретационный принцип** (зафиксирован до маппинга): требование «каждый элемент BAMS трассируется к BDKS» применяется к **доменному знанию**, не к логике метода. Правило «минимум две гипотезы» или шкала уверенности — это метод, собственный слой BAMS; требовать для них доменную трассу означало бы, что методология не имеет права на существование как отдельный уровень (противоречит трёхуровневой модели BDKS-преамбулы). Метод-собственные элементы классифицируются явно, а не маскируются под знание.
