## Шаг 5. Реализованные изменения — верификация и остаточные риски

Все внедрённые CR имеют статус верификации «пройдено» по таблице Шага 4. Остаточные риски:

1. **CR-14 (закрыт 2026-07-02)** — TA исключён из домена решением владельца; `cat: ta` депрецирована на уровне реализации (CLAUDE.md v6.0). Исторических сигналов с категорией нет — миграция данных не требуется. Остаточного риска нет.
2. **CR-19** — направление синхронизации BAMS зафиксировано, но сама синхронизация BAMS (замена копий таксономии/критериев на ссылки) — отдельная будущая правка BAMS; до неё формально существуют две копии (риск управляем декларациями владения CR-03).
3. **CR-01** — наблюдаемые нарратива определены на уровне домена; их операционализация в измеримые величины — задача NIES/реализации. Блокер NIES снят на уровне знания, но NIES потребует уточнения процедур измерения.
4. **Версия**: приращение +0.1 (v1.1) — расширение разделов без изменения структуры и нумерации; соответствует правилу версионирования самого BDKS (Финальный раздел).
