# Логика связей внутри вкладки ТЕОРИЯ

> Создано: 2026-07-09 · Реализация: PR #211 («Вариант C», по итогам обсуждения превью 5 вариантов оформления)

## Общая архитектура раздела

Три панели ТЕОРИИ — не три случайные темы, а последовательность с implicit-логикой, до этой задачи нигде не проговорённой явно:

```
Что такое деньги          →  Семь сетевых эффектов        →  Стратегия DCA
(что это и почему BTC —      (почему это побеждает            (что конкретно делать,
 деньги — определение)        структурно — механизм)           зная 1 и 2 — действие)
```

Порядок панелей уже отражает эту цепочку «понять → почему это работает → как действовать» — панели физически идут в этом порядке в `index.html`, просто раньше этот факт нигде не был явно назван.

## Реализованные точечные связи (Вариант C)

Вместо диаграммы (Вариант D, показан в превью, не реализован) или общей вводной фразы (Вариант A) выбран точечный подход — конкретные пункты аккордеона связаны друг с другом напрямую, не панели целиком.

| # | Откуда | Куда | Логика связи |
|---|---|---|---|
| 1 | Деньги · 05 «Асимметрия» | DCA · 01 «Почему не работает купи дёшево» | Асимметричная ставка (ограниченный риск / неограниченный потенциал) — тот же принцип, который делает попытки поймать точку входа проигрышной стратегией по сравнению с систематическим накоплением |
| 2 | DCA · 04 «Модель PlanB: 1% в BTC» | Деньги · 02 «Шесть свойств денег» | Логика правила «1% портфеля» опирается на дефицитность (одно из шести свойств) как на фундаментальное экономическое обоснование, не просто эвристику |
| 3 | Деньги · 06 «Хозяева и рабы денег» (Бридлав) | Сетевые эффекты · 07 «Мировая резервная валюта» | Тема власти над эмиссией денег (Бридлав, масштаб отдельных монетарных систем) находит продолжение на масштабе всей цивилизации — резервная валюта как предельная форма той же власти |

## Техническая реализация

- Все три панели (`theory-money`, `theory-dca`, `theory-network`) физически находятся в одной вкладке (`tab-theory`) — переход не требует `showTab()`, только `document.getElementById(target).scrollIntoView({behavior:'smooth'})`
- Для статичных пунктов (связи 1 и 2) — блок `.crosslink` вписан прямо в `index.html`
- Для динамического пункта (связь 3, «Хозяева и рабы денег» рендерится из `THEORY_ESSAYS.json` после PR #210) — добавлено поле `crosslink: {text, target_panel, target_label}` в JSON-запись, рендерится через `renderEssayItem()`. Это сохраняет принцип «контент не в HTML» — связь для будущих материалов добавляется тем же способом (запись в JSON), без правки `index.html`
- Визуально: переиспользован уже принятый язык (`bg3`-фон, оранжевая рамка слева) — не новый паттерн с нуля
- **Важно для UX**: связи видны только после раскрытия конкретного пункта аккордеона (они находятся в его теле) — это часть исходного дизайна (Вариант C — точечные связи ВНУТРИ пунктов, не отдельные видимые сверху блоки), не баг
