## Куда эскалировать блокер

Единственное формализованное правило эскалации на 2026-07-02 — из
`docs/IRP_v1.md` §14, только для Sprint 0 Gate: «При несогласии одного из
четырёх — Sprint 0 не открывается до устранения разногласия. Нет права
вето у одного человека, но нет и принципа большинства: каждая роль
покрывает свой домен».

Для остальных блокеров (не Sprint 0 Gate) — принцип ниже предлагается по
аналогии, требует подтверждения:

1. **Первый уровень:** обсудить с тем, кто отвечает за домен (см. RACI,
   Issue #78) — не эскалировать до этого шага
2. **Если не решается:** `[ПЛЕЙСХОЛДЕР — кто следующий уровень]`
3. **SLA на ответ:** `[ПЛЕЙСХОЛДЕР — например, 24 часа в рабочие дни]`
