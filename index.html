Что такое Lightning Network 🔗 LAYER 2

Основная цепь Bitcoin обрабатывает ~7 транзакций в секунду. Весь мир использует ~50 000 в секунду (Visa). Разрыв в 7 000 раз нельзя закрыть, изменив сам блокчейн — это нарушит децентрализацию и безопасность.

Lightning — второй уровень (Layer 2): платёжные каналы, которые работают поверх блокчейна, не записывая каждую транзакцию в цепочку. Два участника открывают канал одной on-chain транзакцией, совершают тысячи мгновенных платежей между собой, затем закрывают канал — итог фиксируется в блокчейне.

Результат: комиссии в доли цента, скорость — миллисекунды, пропускная способность — миллионы транзакций в секунду теоретически.

// Жизненный цикл канала
1. Открытие → 1 on-chain tx · депозит в мультисиг
2. Платежи → ∞ off-chain · мгновенно, ~0 комиссий
3. Закрытие → 1 on-chain tx · итоговые балансы
Как работает маршрутизация МЕХАНИКА
🔗 Платёжные каналы ▼

Канал — это мультисиг-кошелёк 2-of-2 в блокчейне. Оба участника вносят BTC. Каждый платёж — это подписанное состояние (commitment transaction), которое можно предъявить в сеть в любой момент.

Хитрость: стороны всегда держат актуальную подписанную транзакцию, но публикуют только финальную. Обман карается: если один попытается транслировать старое состояние — второй может забрать весь депозит через revocation key.

🌐 Сеть каналов и HTLC ▼

Не нужно открывать канал с каждым. Если Алиса → Боб → Кэрол, платёж маршрутизируется через Боба. Для защиты используются HTLC (Hash Time-Locked Contracts): Боб получает комиссию только если выполнит условие, иначе деньги возвращаются автоматически.

Маршрутизация onion-шифрованная (как Tor): Боб знает только предыдущий и следующий узел, не видит ни отправителя, ни получателя целиком.

💧 Проблема ликвидности ▼

Канал имеет две стороны: исходящая ликвидность (сколько можешь отправить) и входящая (сколько можешь получить). Сумма обеих сторон равна депозиту канала.

Главная сложность Lightning: получить входящую ликвидность, то есть чтобы кто-то открыл канал к тебе или ты занял место на их стороне. Без входящей ликвидности принимать платежи нельзя.

Как подключиться к Lightning ПРАКТИКА
📱 Путь 1 — Кошелёк-кастодиан (просто, без узла) ▼

Самый быстрый старт. Приложение само держит каналы — тебе нужен только кошелёк. Ключи хранит сервис, не ты. Подходит для небольших сумм и знакомства с сетью.

Strike // США/Европа, биржа + Lightning
Wallet of Satoshi // самый простой, iOS/Android
CashApp // США, встроенный Lightning
Muun // гибрид on-chain + LN

⚠️ Not your keys — not your coins. Держи только то, что готов потерять.

🔑 Путь 2 — Non-custodial кошелёк (твои ключи, без узла) ▼

Ключи хранишь ты, но управление каналами автоматизировано. Компромисс: суверенность без сложности запуска ноды. Каналы открывает автоматически встроенный LSP (Lightning Service Provider).

Phoenix // лучший баланс простоты и суверенности
Breez // подкасты (Podcasting 2.0), POS
Zeus // подключается к своей ноде или LSP

Phoenix (ACINQ) берёт комиссию ~0.4% за создание/изменение канала (splicing).

🖥️ Путь 3 — Своя нода (полный контроль) ▼

Запускаешь собственный Bitcoin + Lightning узел. Полное самохранение: ты маршрутизатор, никаких LSP. Требуется: мини-ПК (Raspberry Pi 4 / NUC), 1–2 ТБ SSD, ~неделя на синхронизацию Bitcoin Core.

Ограничения и риски ОГОВОРКИ

Нода должна быть онлайн. Для получения платежей узел должен быть доступен. Если нода падает — входящие платежи не проходят.

Потеря данных канала опаснее, чем в on-chain. Если потерять state канала и переиздать старую транзакцию — контрагент может забрать весь депозит. Используй Static Channel Backups (SCB).

Максимальный размер платежа ограничен ликвидностью канала. Большие суммы (десятки BTC) через Lightning не провести — нужен on-chain.

Экспериментальный статус. Lightning активно развивается. Splicing, taproot channels, Bolt 12 — стандарты ещё меняются. Держи небольшие суммы.

MSTR — обыкновенная акция Strategy (бывш. MicroStrategy). Фактически работает как инструмент BTC-экспозиции с левереджем: компания конвертирует привлечённый капитал в биткоин и держит его на балансе. Стоит ниже STRC в капитальной структуре — при ликвидации держатели MSTR получают выплаты после привилегированных акционеров.
КАК РАБОТАЕТ
Strategy выпускает STRC/облигации → привлекает USD → покупает BTC → BTC растёт → MSTR торгуется с премией к NAV → можно выпускать новые инструменты на выгодных условиях. Петля самоусиления работает только пока акция стоит дороже BTC-резерва.
NAV — СТОИМОСТЬ ЧИСТЫХ АКТИВОВ
NAV (Net Asset Value) — справедливая стоимость компании если продать все активы и погасить все долги. Для Strategy это почти синоним стоимости BTC-резерва: 847 363 BTC × цена BTC = NAV. Когда MSTR стоит дороже NAV — это премия. Когда дешевле — дисконт.
КАК СЧИТАЕТСЯ ▶
ФОРМУЛА
BTC NAV = кол-во BTC × цена BTC
Чистый NAV = BTC NAV − долг − обязательства
Мультипликатор = цена акции ÷ (Чистый NAV ÷ акции в обращении)
Пример (июнь 2026): 847 363 BTC × $59 000 = $49,9 млрд BTC NAV. Минус долг ~$8,2 млрд и привилегированные обязательства ~$4,1 млрд = Чистый NAV ~$37,6 млрд. Акций в обращении ~242 млн → NAV на акцию ~$155. MSTR торгуется по $92 → мультипликатор 0.59×.
ГДЕ БЕРУТСЯ ДАННЫЕ ▶
BTC РЕЗЕРВ SEC-раскрытие (форма 8-K после каждой покупки) + on-chain верификация через Arkham Intelligence. Данные публичны и верифицируемы.
ЦЕНА BTC Spot-цена с бирж (Coinbase, Binance). Разные источники используют разные эталонные цены — отсюда небольшие расхождения в цифрах NAV у разных аналитиков.
ДОЛГ Квартальный отчёт 10-Q / 10-K в SEC. Включает конвертируемые облигации, обеспеченные кредиты. Обновляется раз в квартал — между отчётами данные могут устаревать.
АГРЕГАТОРЫ SaylorTracker.com (неофициальный, обновляется в реальном времени), Coinglass, BitcoinTreasuries.net. Для точного расчёта всегда сверяться с первичными SEC-документами.

ПРЕМИЯ > 1×
Рынок платит за левередж и удобство доступа к BTC через акцию. Исторически 1.5–3× в 2020–2024. Позволяет Strategy выпускать новые инструменты выгодно.
ДИСКОНТ < 1×

Почему сейчас дисконт: запуск спотовых BTC-ETF в январе 2024 убил главный аргумент в пользу премии — «нет другого способа получить BTC через брокерский счёт». Одновременно дивидендные обязательства выросли 4× — рынок закладывает риск что их придётся покрывать за счёт BTC.

Что такое деньги ТОНИ · 2019
Деньги — самый ликвидный товар на планете. Но большинство людей никогда не задумывались как они устроены. Это напрямую влияет на любое финансовое решение.
01 Три функции денег ▼

Деньги — инструмент для организации торговли. Чтобы выполнять эту роль, товар должен одновременно быть:

СРЕДСТВО СБЕРЕЖЕНИЯ
Хорошо хранить собственную ценность. Без этого никто не захочет держать деньги — они начнут тратить немедленно, экономика деградирует к бартеру.
СРЕДСТВО ОБМЕНА
Быть востребованным при торговле. Чем больше участников готовы принять этот товар — тем лучшие это деньги.
ЕДИНИЦА ИСЧИСЛЕНИЯ
Быть мерилом стоимости всего остального. Это финальная стадия — когда большинство участников измеряют цены именно в этом товаре.
Валюты (доллар, евро) выполняют только 2 из 3 функций — средство обмена и единица исчисления. Средством сбережения они не являются: их можно напечатать в любом количестве.
↳ «Средство обмена» — не теоретическое допущение, а функция, которая либо подтверждается, либо нет реальным потребительским принятием Сетевые эффекты · 03
02 Шесть свойств денег ▼
РЕДКОСТЬ
Основа ценности. Без редкости товар не может сохранять стоимость. Золото — редко природно. Bitcoin — редко математически: ровно 21 млн, не больше никогда.
ВЗАИМОЗАМЕНЯЕМОСТЬ
Каждая единица идентична любой другой. 1 BTC = 1 BTC. Бриллианты не взаимозаменяемы — каждый уникален, поэтому не могут быть деньгами.
ДЕЛИМОСТЬ
Bitcoin делится до 8 знаков после запятой: 1 BTC = 100 000 000 сатоши. Золото физически неудобно делить для мелких покупок.
ТРАНСПОРТАБЕЛЬНОСТЬ
474 тонны золота из ФРС США в Германию транспортировались 4 года. Транзакция Bitcoin на $1 млрд заняла несколько часов и стоила $70.
ПРОЧНОСТЬ
Цифровой актив не изнашивается. Bitcoin можно передавать бесконечное количество раз без деградации.
ВЕРИФИЦИРУЕМОСТЬ
Любой может в любой момент проверить сколько BTC существует, кому принадлежит и когда были созданы — без обращения к банку или регулятору. Это возможно благодаря децентрализации: у сети нет единой точки доверия, проверку не нужно принимать на слово от посредника. Золото — нет: мы не знаем сколько золота в астероиде Psyche 16.
03 История денег: от бартера к Bitcoin ▼
БАРТЕР
Прямой обмен товарами. Проблема: нужно двойное совпадение желаний. Не масштабируется — количество цен растёт как n² от числа участников.
КАМНИ ЯП
Остров Яп: огромные каменные диски как деньги. Никуда не двигались — менялась лишь запись о владельце в памяти сообщества. Та же логика что у Bitcoin.
ЗОЛОТО
~5000 лет денежного стандарта. Высокое S2F (запас/поток = 62). Проблема: сложно делить и транспортировать. Рим начал разбавлять монеты медью — первая «инфляция».
ФИАТ
1971 — Nixon shock: доллар отвязан от золота. Fiat с латыни — «да будет так». Дюжина чиновников решает сколько напечатать самого востребованного товара в мире. Зимбабве, Венесуэла — закономерный результат.
BITCOIN
Первый актив с исчисляемой цифровой редкостью. Делится на 100 млн сатоши. Транспортируется мгновенно. Не изнашивается. Верифицируется публично. S2F удваивается каждые ~4 года через халвинг.

05 Асимметрия ▼

Bitcoin молод — 17 лет против 5000 лет золота. Общая капитализация ~$1.2 трлн против десятков триллионов мировых денег. Активных пользователей — несколько сотен миллионов из 8 млрд.

Если Bitcoin действительно лучшие деньги — а по свойствам это так — то текущая капитализация составляет лишь малую долю от потенциальной. Соотношение возможного роста к возможным потерям асимметрично в пользу роста.

Сетевые эффекты Bitcoin усиливают друг друга по нарастающей — каждый новый участник делает сеть ценнее для всех остальных. Любому конкуренту нужно превзойти все семь одновременно.
01 Спекуляция ▼

Bitcoin — новый класс криптографически подкреплённых активов с потенциалом роста и высокой волатильностью. Это привлекает спекулянтов с высокой толерантностью к риску. Они создают первичную ликвидность и ценовое открытие — без этого не запускаются остальные эффекты.

Спекуляция → ликвидность → цена → стимул для майнеров → безопасность сети
↳ Волатильность, которую создают спекулянты — источник той самой асимметричной ставки, а не просто её сопровождение Деньги · 05
02 Принятие бизнесами ▼

Бизнесы принимают Bitcoin потому что это увеличивает маржу: нет возвратных платежей, нет комиссий платёжных систем (2–3%), нет зависимости от банков. Каждый новый мерчант расширяет полезность сети для потребителей.

2026: корпоративные казначейства как следующий уровень — бизнес не просто принимает BTC, но и держит его на балансе
03 Принятие потребителями ▼

Потребители получают доступ к товарам и услугам которые иначе недоступны: трансграничные переводы без банков, покупки несмотря на санкции и эмбарго, микроплатежи. Bitcoin повышает экономическую эффективность особенно в нишах где традиционная финансовая система неэффективна.

04 Безопасность ▼

Принятие → рост цены → больше майнеров → выше хешрейт → дороже атака. Децентрализованный необратимый реестр — форма тройного бухучёта: дебет, кредит и публичное подтверждение в сети. Каждый новый участник делает систему безопаснее для всех.

Хешрейт июнь 2026: ~900 EH/s — исторический максимум несмотря на падение цены
05 Предпочтение разработчиков ▼

Bitcoin — простая и предсказуемая сеть с элементарными правилами и открытым кодом. Децентрализованная природа допускает инновации без разрешения. Разработчики тянутся к Bitcoin потому что здесь самая глубокая ликвидность, самая высокая безопасность и самый консервативный протокол — именно то что нужно для финансовой инфраструктуры.

06 Финансирование ▼

Bitcoin постепенно поглощает функции традиционных банков: денежные переводы, микроплатежи, p2p-кредитование, торговля активами. Институциональный капитал входит через ETF, корпоративные казначейства, суверенные резервы. Этот процесс необратим — деньги, однажды вошедшие в Bitcoin-инфраструктуру, не возвращаются в старую систему.

Стратегия DCA SVETSKI · PLANB · 2020
Dollar Cost Averaging — покупать фиксированную сумму через равные промежутки времени вне зависимости от цены. Убирает эмоции, сглаживает волатильность, усредняет цену входа.
01 Почему не работает «купи дёшево» ▼

«Покупай дёшево, продавай дорого» — мантра, которой следуют все. Менее 10% участников рынка делают это регулярно.

Причина: рынки — это отражение эмоций большинства. Когда всё растёт — люди покупают дороже из жадности. Когда падает — продают на дне из страха.

ЭМОЦИОНАЛЬНЫЙ ИНВЕСТОР
Покупает на хайпе → держит на просадке → продаёт на панике → пропускает восстановление. Итог: минус при растущем активе.
DCA-ИНВЕСТОР
Покупает каждый месяц → на просадке получает больше BTC за те же деньги → на росте получает меньше. Эмоции исключены из уравнения.
02 Как работает DCA ▼

Вместо одного крупного входа — серия равных покупок через фиксированные интервалы. Цена каждой покупки разная, итоговая средняя цена входа сглаживается.

Месяц Цена BTC Сумма Куплено BTC
Янв$100K$5000.005
Фев$80K$5000.00625
Мар$60K$5000.00833
Апр$90K$5000.00556
Потрачено $2000 → куплено 0.025 BTC → средняя цена $80K Одноразовая покупка в январе по $100K: те же $2000 = 0.020 BTC
03 Шесть преимуществ ▼
МЕНЬШЕ ПРИ РОСТЕ, БОЛЬШЕ ПРИ ПАДЕНИИ
Автоматически покупаешь больше BTC когда дёшево и меньше когда дорого — без анализа.
НИЖЕ СРЕДНЯЯ ЦЕНА ВХОДА
Итоговая средняя стоимость BTC в портфеле ниже чем при единоразовой покупке на пике.
БЕЗ ЭМОЦИЙ
Решение принимается один раз в начале. Дальше — автоматика. FOMO и паника исключены.
НЕ НУЖНА КРУПНАЯ СУММА
Можно начать с $50 в месяц. Размер позиции наращивается постепенно.
НЕТ СТРЕССА ОТ ТАЙМИРОВАНИЯ
Не нужно угадывать «правильный момент». Рынок сам усредняет за тебя через время.
РАБОТАЕТ НА ВОЛАТИЛЬНЫХ АКТИВАХ
Чем выше волатильность — тем эффективнее DCA. Bitcoin идеален для этой стратегии.

mNAV — механика BTC-казначейств ИНСТИТУЦИОНАЛЬНЫЙ СЛОЙ
mNAV (market Net Asset Value) — ключевой показатель для оценки публичных BTC-казначейств. Определяет режим работы всей модели: аккреция или разводнение. Понимание mNAV объясняет почему Strategy, Metaplanet и другие делают те или иные ходы.
01 Что такое mNAV ▼

mNAV показывает насколько рынок оценивает компанию относительно стоимости её Bitcoin-резервов.

ФОРМУЛА
mNAV = (Капитализация + Долг − Кэш) ÷ Стоимость BTC
mNAV > 1x — ПРЕМИЯ
Рынок платит больше чем стоит BTC внутри. Инвесторы верят в модель и будущие покупки. Любая эмиссия акций аккретивна — компания продаёт акции дороже чем стоит её BTC и покупает ещё.
mNAV = 1x — ПАРИТЕТ
Акция стоит ровно столько сколько BTC внутри. Нейтральная точка — ни аккреция ни разводнение.
mNAV < 1x — ДИСКОНТ
Акция дешевле чем BTC внутри. Эмиссия разводняет — компания продаёт $1 BTC за $0.90. Та же машина работает в обратную сторону. Strategy в июне 2026: 0.83x.
02 Почему возникает премия ▼

Инвесторы платят премию сверх стоимости BTC по нескольким причинам одновременно:

ЛИКВИДНОСТЬ И ДОСТУПНОСТЬ
Пенсионные фонды, страховые компании и многие институты не могут напрямую купить BTC — но могут купить акцию на бирже. Акция = регулируемый доступ к BTC-экспозиции.
ВСТРОЕННОЕ КРЕДИТНОЕ ПЛЕЧО
Компания берёт долг и покупает BTC. Акционер получает экспозицию больше чем 1:1 к BTC. При росте BTC акция растёт быстрее — встроенный леверидж без маржин-колла.
ДОВЕРИЕ К МЕНЕДЖМЕНТУ
Рынок платит за то что команда будет продолжать покупать BTC и увеличивать BTC per share. Это опцион на будущие покупки — он стоит денег.
НАРРАТИВНАЯ ПРЕМИЯ
Strategy в 2021–2024 торговалась по 2–3x mNAV. Часть премии — вера что модель станет стандартом. Когда нарратив ослабевает — премия сжимается.
03 Машина аккреции: как работает при премии ▼

При mNAV > 1x включается маховик накопления:

ЦИКЛ АККРЕЦИИ (mNAV > 1x)
1. Акция торгуется с премией к BTC
2. Компания выпускает новые акции по рыночной цене
3. На привлечённые деньги покупает BTC
4. BTC per share растёт — премия сохраняется
5. → вернуться к шагу 1
Strategy провела этот цикл сотни раз в 2021–2024. При каждой эмиссии по 2x mNAV она фактически покупала BTC вдвое дешевле рынка за счёт инвесторов платящих премию.
04 Дисконт: когда машина ломается ▼

При mNAV < 1x та же механика работает против акционеров:

ЦИКЛ РАЗВОДНЕНИЯ (mNAV < 1x)
1. Акция торгуется дешевле BTC внутри
2. Любая эмиссия продаёт $1 BTC за $0.90
3. BTC per share падает — дисконт усиливается
4. Новая эмиссия ещё более разводняет
5. → спираль вниз
BUYBACK КАК ВЫХОД
При mNAV < 1x обратный выкуп акций = покупка BTC по дисконту. Metaplanet рассматривал buyback при mNAV 0.90x в июне 2026 именно по этой логике.
ПРОДУКТОВЫЙ БИЗНЕС КАК ВЫХОД
Metaplanet купил Siiibo Securities: комиссионный доход от BTC-продуктов не зависит от mNAV. Это второй движок независимый от цикла премии.
05 BTC Yield — главный KPI ▼

BTC Yield — не процент и не дивиденд. Это рост BTC на одну акцию в процентах за период.

ФОРМУЛА
BTC Yield = (BTC per share сейчас − BTC per share раньше) ÷ BTC per share раньше

Положительный BTC Yield означает что на каждую акцию приходится больше BTC чем раньше — даже если цена BTC упала. Это единственное что важно для долгосрочного держателя акции казначейства.

Компания BTC mNAV BTC Yield
Strategy847 3630.83x~15% YTD
Metaplanet40 177<1x2.8% Q1
mNAV может быть ниже 1x, а BTC Yield — положительным одновременно. Strategy в июне 2026: дисконт к NAV, но BTC per share растёт. Это разные измерения разных вещей.
06 Как читать mNAV при анализе сигналов ▼
ЭМИССИЯ АКЦИЙ → ПРОВЕРЬ mNAV
При mNAV > 1x — бычий сигнал, аккреция. При mNAV < 1x — медвежий, разводнение. Контекст мгновенно меняет интерпретацию одного и того же действия.
ПОКУПКА BTC КОМПАНИЕЙ → СМОТРИ ИСТОЧНИК ФИНАНСИРОВАНИЯ
Долг при mNAV > 1x = аккретивно. Долг при mNAV < 1x = риск. Кэш из операционного дохода = нейтрально при любом mNAV.
BUYBACK → СИГНАЛ ДНА ИЛИ УВЕРЕННОСТИ
Компания выкупает акции при mNAV < 1x = покупает собственный BTC по дисконту. Сильный сигнал уверенности менеджмента в долгосрочной стоимости.
mNAV ВСЕГО СЕКТОРА → ТЕРМОМЕТР НАРРАТИВА
Если Strategy, Metaplanet и другие одновременно уходят ниже 1x — рынок разочарован моделью в целом. Если только один — проблема конкретной компании.

⛏ Proof-of-Work ▼

Чтобы добавить блок, майнер находит nonce — число, при котором SHA-256 хэш начинается с нужного количества нулей. Это чистый перебор: миллиарды хэшей в секунду.

Сеть автоматически регулирует сложность каждые 2016 блоков (~2 недели), чтобы среднее время блока оставалось 10 минут независимо от мощности майнеров.

// Упрощённый PoW
while True:
  hash = SHA256(block + nonce)
  if hash.startswith("0000"):
    return nonce // Найдено!
  nonce += 1
🔐 SECP256k1 криптография ▼

Bitcoin использует эллиптические кривые: y² = x³ + 7. Приватный ключ — случайное 256-битное число. Публичный ключ — точка на кривой.

Умножение точки — односторонняя функция: из приватного легко получить публичный, но не наоборот. Подпись транзакции доказывает владение без раскрытия ключа.

📜 Скрипты Bitcoin ▼

Каждый UTXO заперт скриптом блокировки. Транзакция предоставляет скрипт разблокировки. Это стэковый язык программирования.

// P2PKH — стандартный адрес 1...
Lock: OP_DUP OP_HASH160 <pubKeyHash>
        OP_EQUALVERIFY OP_CHECKSIG
Unlock: <sig> <pubKey>

Типы: P2PKH (1...) · P2SH (3..., мультисиг) · SegWit (bc1...) · Taproot (bc1p..., приватность)

⚡ Lightning Network ▼

Основная цепь обрабатывает ~7 транзакций в секунду. Lightning — второй уровень: платёжные каналы вне блокчейна.

Открываешь канал (1 on-chain транзакция) → тысячи мгновенных платежей → закрываешь (1 on-chain). Комиссии — доли цента. Скорость — мгновенно.

Это обзор протокола. LIGHTNING → — маршрутизация, 3 способа подключения, риски

⛓ Блокчейн ▼

Блокчейн — публичная база данных всех транзакций. Каждый блок содержит транзакции, временну́ю метку и хэш предыдущего блока. Изменить один блок — значит пересчитать всю последующую цепочку.

// Структура блока

🔑 Ключи и адреса ▼

Приватный ключ — секретное 256-битное число. Контролирует монеты.

Публичный ключ генерируется из приватного через эллиптические кривые. Обратная операция вычислительно невозможна.

Адрес — хэш публичного ключа.

// Цепочка генерации
privKey → SECP256k1 → pubKey
pubKey → SHA256 + RIPEMD160 → addr

🪙 Модель UTXO ▼

Bitcoin не хранит «баланс». Вместо этого — UTXO (Unspent Transaction Outputs): набор «монеток», адресованных твоему ключу. При трате уничтожаешь старые UTXO и создаёшь новые. Разница уходит майнеру как комиссия.
