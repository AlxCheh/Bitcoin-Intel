Блокчейн СТРУКТУРА

Блокчейн — публичная база данных всех транзакций. Это книга учёта, в которую все могут заглянуть, но никто не может изменить прошлое.

Каждый блок содержит транзакции, временну́ю метку и хэш предыдущего блока. Изменить один блок — значит пересчитать всю последующую цепочку.

// Структура блока
Block #850001 {
  prev_hash "0000000000abc4f2..."
  merkle_root "d4e5f6a2..."
  timestamp 2024-04-20 14:32:07
  nonce 3,942,891
  tx_count 2,834
}
Ключи и адреса КРИПТОГРАФИЯ

Приватный ключ — секретное 256-битное число. Тот, кто знает ключ, контролирует монеты.

Публичный ключ генерируется из приватного через эллиптические кривые. Обратная операция вычислительно невозможна.

Адрес — хэш публичного ключа. Его даёшь другим для получения платежей.

// Цепочка генерации
privKey → SECP256k1 → pubKey
pubKey → SHA256 + RIPEMD160 → addr

⚡ Lightning Network
Что такое Lightning Network 🔗 LAYER 2

Основная цепь Bitcoin обрабатывает ~7 транзакций в секунду. Весь мир использует ~50 000 в секунду (Visa). Разрыв в 7 000 раз нельзя закрыть, изменив сам блокчейн — это нарушит децентрализацию и безопасность.

Lightning — второй уровень (Layer 2): платёжные каналы, которые работают поверх блокчейна, не записывая каждую транзакцию в цепочку. Два участника открывают канал одной on-chain транзакцией, совершают тысячи мгновенных платежей между собой, затем закрывают канал — итог фиксируется в блокчейне.

Результат: комиссии в доли цента, скорость — миллисекунды, пропускная способность — миллионы транзакций в секунду теоретически.

// Жизненный цикл канала
1. Открытие → 1 on-chain tx · депозит в мультисиг
2. Платежи → ∞ off-chain · мгновенно, ~0 комиссий
3. Закрытие → 1 on-chain tx · итоговые балансы
Как работает маршрутизация МЕХАНИКА
🔗 Платёжные каналы ▼

Канал — это мультисиг-кошелёк 2-of-2 в блокчейне. Оба участника вносят BTC. Каждый платёж — это подписанное состояние (commitment transaction), которое можно предъявить в сеть в любой момент.

Хитрость: стороны всегда держат актуальную подписанную транзакцию, но публикуют только финальную. Обман карается: если один попытается транслировать старое состояние — второй может забрать весь депозит через revocation key.

🌐 Сеть каналов и HTLC ▼

Не нужно открывать канал с каждым. Если Алиса → Боб → Кэрол, платёж маршрутизируется через Боба. Для защиты используются HTLC (Hash Time-Locked Contracts): Боб получает комиссию только если выполнит условие, иначе деньги возвращаются автоматически.

Маршрутизация onion-шифрованная (как Tor): Боб знает только предыдущий и следующий узел, не видит ни отправителя, ни получателя целиком.

💧 Проблема ликвидности ▼

Канал имеет две стороны: исходящая ликвидность (сколько можешь отправить) и входящая (сколько можешь получить). Сумма обеих сторон равна депозиту канала.

Главная сложность Lightning: получить входящую ликвидность, то есть чтобы кто-то открыл канал к тебе или ты занял место на их стороне. Без входящей ликвидности принимать платежи нельзя.

Как подключиться к Lightning ПРАКТИКА
📱 Путь 1 — Кошелёк-кастодиан (просто, без узла) ▼

Самый быстрый старт. Приложение само держит каналы — тебе нужен только кошелёк. Ключи хранит сервис, не ты. Подходит для небольших сумм и знакомства с сетью.

Strike // США/Европа, биржа + Lightning
Wallet of Satoshi // самый простой, iOS/Android
CashApp // США, встроенный Lightning
Muun // гибрид on-chain + LN

⚠️ Not your keys — not your coins. Держи только то, что готов потерять.

🔑 Путь 2 — Non-custodial кошелёк (твои ключи, без узла) ▼

Ключи хранишь ты, но управление каналами автоматизировано. Компромисс: суверенность без сложности запуска ноды. Каналы открывает автоматически встроенный LSP (Lightning Service Provider).

Phoenix // лучший баланс простоты и суверенности
Breez // подкасты (Podcasting 2.0), POS
Zeus // подключается к своей ноде или LSP

Phoenix (ACINQ) берёт комиссию ~0.4% за создание/изменение канала (splicing).

🖥️ Путь 3 — Своя нода (полный контроль) ▼

Запускаешь собственный Bitcoin + Lightning узел. Полное самохранение: ты маршрутизатор, никаких LSP. Требуется: мини-ПК (Raspberry Pi 4 / NUC), 1–2 ТБ SSD, ~неделя на синхронизацию Bitcoin Core.

// Готовые сборки (проще всего)

RaspiBlitz // open-source, более хардкорный
Start9 // упор на приватность
// Реализации Lightning-ноды
LND // Lightning Labs, самый популярный
CLN // Core Lightning (Blockstream)
Eclair // ACINQ, Java/Scala

Ограничения и риски ОГОВОРКИ

Нода должна быть онлайн. Для получения платежей узел должен быть доступен. Если нода падает — входящие платежи не проходят.

Потеря данных канала опаснее, чем в on-chain. Если потерять state канала и переиздать старую транзакцию — контрагент может забрать весь депозит. Используй Static Channel Backups (SCB).

Максимальный размер платежа ограничен ликвидностью канала. Большие суммы (десятки BTC) через Lightning не провести — нужен on-chain.

Экспериментальный статус. Lightning активно развивается. Splicing, taproot channels, Bolt 12 — стандарты ещё меняются. Держи небольшие суммы.

⛏ Proof-of-Work ▼

Чтобы добавить блок, майнер находит nonce — число, при котором SHA-256 хэш начинается с нужного количества нулей. Это чистый перебор: миллиарды хэшей в секунду.

Сеть автоматически регулирует сложность каждые 2016 блоков (~2 недели), чтобы среднее время блока оставалось 10 минут независимо от мощности майнеров.

// Упрощённый PoW
while True:
  hash = SHA256(block + nonce)
  if hash.startswith("0000"):
    return nonce // Найдено!
  nonce += 1
🪙 Модель UTXO ▼

Bitcoin не хранит балансы. Существуют UTXO — неизрасходованные выходы транзакций. Твой баланс — сумма всех UTXO на твоих адресах.

При трате уничтожаешь старые UTXO и создаёшь новые. Разница уходит майнеру как комиссия. Транзакции можно проверять параллельно — это делает Bitcoin эффективнее Ethereum на базовом уровне.

🔐 SECP256k1 криптография ▼

Bitcoin использует эллиптические кривые: y² = x³ + 7. Приватный ключ — случайное 256-битное число. Публичный ключ — точка на кривой.

Умножение точки — односторонняя функция: из приватного легко получить публичный, но не наоборот. Подпись транзакции доказывает владение без раскрытия ключа.

📜 Скрипты Bitcoin ▼

Каждый UTXO заперт скриптом блокировки. Транзакция предоставляет скрипт разблокировки. Это стэковый язык программирования.

// P2PKH — стандартный адрес 1...
Lock: OP_DUP OP_HASH160 <pubKeyHash>
        OP_EQUALVERIFY OP_CHECKSIG
Unlock: <sig> <pubKey>

Типы: P2PKH (1...) · P2SH (3..., мультисиг) · SegWit (bc1...) · Taproot (bc1p..., приватность)

⚡ Lightning Network ▼

Основная цепь обрабатывает ~7 транзакций в секунду. Lightning — второй уровень: платёжные каналы вне блокчейна.

Открываешь канал (1 on-chain транзакция) → тысячи мгновенных платежей → закрываешь (1 on-chain). Комиссии — доли цента. Скорость — мгновенно.
